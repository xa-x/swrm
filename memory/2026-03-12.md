## Project: Swrm - AI Agent Deployment Platform

### What We Built
- **Swrm**: Platform for deploying AI agents from mobile app
- **Repository**: https://github.com/xa-x/swrm
- **Architecture**: Expo Native + Convex Backend + Fly.io Machines

### Tech Stack
- **Frontend**: Expo Router (Native), Next.js (Web - placeholder)
- **Backend**: Convex (real-time database + functions)
- **Auth**: Clerk
- **Containers**: Fly.io Machines running ZeroClaw
- **Cost**: ~$0.50-2/agent/month

### Monorepo Structure
```
swrm/
├── apps/
│   ├── native/          # Expo mobile app
│   └── web/             # Next.js (placeholder)
├── packages/
│   ├── backend/         # Convex backend
│   │   └── convex/      # Schema, functions, crons
│   ├── shared/          # Shared types & utilities
│   ├── eslint-config/
│   └── typescript-config/
```

### Key Decisions
1. **Convex over Bun+Hono**: Real-time subscriptions, built-in dashboard, simpler ops
2. **BYOK Model**: Users bring their own API keys (zero AI cost to us)
3. **ZeroClaw containers**: 8.8MB binary, <5MB memory, perfect for this
4. **Fly.io Machines**: Per-agent containers, $0.0015/hr, auto-scaling

### Environment Variables
- `EXPO_PUBLIC_CONVEX_URL`: Convex backend URL
- `EXPO_PUBLIC_CLERK_PUBLISHABLE_KEY`: Clerk auth key
- `FLY_API_TOKEN`: Fly.io API token (set in Convex env)
- `FLY_APP_NAME`: swrm-agents

### Screens Implemented
- **Home**: Card grid of agents (real-time via Convex)
- **Create**: 4-step wizard (name → provider → personality → skills)
- **Chat**: Markdown support, real-time messages
- **Agent Details**: Status, actions, usage stats
- **Broadcast**: Send to multiple agents at once

### Monorepo Setup (Based on Shasha App)
- Metro config with `watchFolders` for workspace root
- Babel module-resolver for `@swrm/backend`, `@swrm/shared`
- TypeScript path aliases
- Workspace dependencies in package.json

### Convex Functions
- `agents.ts`: CRUD + start/stop/restart
- `chat.ts`: Messages + real-time history
- `usage.ts`: Token/cost tracking
- `docker.ts` / `fly.ts`: Container management
- `crons.ts`: Cleanup, monitoring

### Features
- Real-time sync (Convex subscriptions)
- Offline-first (Expo-SQLite cache)
- Push notifications (Expo)
- Docker actions logged
- Usage tracking per agent
